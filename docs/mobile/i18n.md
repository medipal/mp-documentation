- Plugin: `@nuxtjs/i18n` 9.3.1
- Strategy: `no_prefix` ‚Äî locale is NOT in the URL path
- Default locale: `en_GB`

## Usage in Components

```ts
// Composition API
const { t, locale } = useI18n();

// In template
{
  {
    t("home.questionnaire.start_button");
  }
}
```

## Adding Translations

1. Add the key to **all 10** locale JSON files
2. Use descriptive, namespaced keys: `section.subsection.element`
3. Avoid HTML in translations; if required, `escapeHtml: false` is already configured

::: warning Key naming convention
Use `snake_case` with dot-separated namespaces: `home.questionnaire.start_button`
Never use camelCase or hyphens in i18n keys.
:::

## Locale Codes ‚Äî Underscores, Not Hyphens

::: danger Critical convention
Locale codes use **underscores**: `en_GB`, `de_DE`, `ar_SA`.

The locale objects have a separate `language` property (`en-GB`) used only for the HTML `lang` attribute. Everywhere else ‚Äî Pinia store, i18n plugin, locale filenames, comparisons ‚Äî use the underscore format.
:::

## Locale Persistence

1. On startup: resolved from persisted user preference ‚Üí device language ‚Üí fallback `en_GB`
2. Stored in `device.userLocale` (Pinia) and persisted via `useLocalConfig()`
3. Synced to the `@nuxtjs/i18n` plugin in `app/pages/index.vue` on locale changes

## RTL Support

Arabic (`ar_SA`) is the only RTL locale.

- The `<html dir="rtl">` attribute is set automatically by the i18n plugin
- Use logical CSS properties (`margin-inline-start` instead of `margin-left`) in components
- Always test new layout changes with Arabic locale active
